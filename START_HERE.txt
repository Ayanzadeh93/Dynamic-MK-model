QUICK START (3 Steps):
---------------------

1. Install Requirements:
   > pip install -r requirements.txt

2. Verify Setup:
   > python test_setup.py

3. Start Training:
   > python train_kfold.py

That's it! Training will run automatically with 20-fold cross validation.

================================================================================

WHAT YOU GET:
-------------

After training completes, you'll have:
  - 20 trained model checkpoints (one per fold)
  - Comprehensive evaluation metrics (accuracy, precision, recall, F1, ROC-AUC, etc.)
  - Beautiful visualizations (confusion matrices, ROC curves, training history)
  - Aggregate statistics across all folds (mean +/- std)
  - Ready-to-use models for inference

All results saved in: results_squeezenet_kfold20/

================================================================================

TRAINING TIME:
--------------

Expected training time for 20 folds:
  - With GPU (CUDA):  2-4 hours
  - With CPU:         10-20 hours

Progress is shown in real-time with:
  - Progress bars for each epoch
  - Loss and accuracy updates
  - Best model notifications

CONFIGURATION:
  Open train_kfold.py and modify the 'config' dictionary:
    - num_folds: 20          (change to 5 for faster testing)

================================================================================

COMMON ADJUSTMENTS:
-------------------

If training is TOO SLOW:
  In train_kfold.py config, change:
    'num_folds': 5,        (instead of 20)
    'num_epochs': 30,      (instead of 50)

If you get OUT OF MEMORY error:
  In train_kfold.py config, change:
    'batch_size': 8,       (instead of 16)
    'freeze_backbone': True,

For BETTER ACCURACY:
  In train_kfold.py config, try:
    'num_epochs': 100,
    'learning_rate': 0.0001,  (or 0.01)

================================================================================

TROUBLESHOOTING:
----------------

Problem: "No module named 'torch'"
Solution: pip install -r requirements.txt

Problem: "CUDA out of memory"
Solution: Reduce batch_size to 8 or 4 in config

Problem: Training too slow
Solution: Use GPU, or reduce num_folds to 5

Problem: No improvement in accuracy
Solution: Try different learning_rate (0.0001 or 0.01)

REUSABLE COMPONENTS:

The evaluation_metrics.py module is designed to work with ANY binary
